NURA current PCB pinmap summary

Scope
- Teensy 4.1 avionics PCB pin assignment from the current board bring-up notes.
- All sensor/module VCC pins use the Teensy 3.3 V rail unless the hardware schematic says otherwise.
- All GND pins share the common avionics ground.
- Pin ownership lives in include/board_pinmap.h. This file is the human wiring checklist.

Pin ownership check
- Pyro 1 is the drogue channel and uses D28/D29; Pyro 2 is the main channel and has been rerouted to the D35/D38 output pair.
- Pyro 2 gpio1/gpio2 order must be confirmed by continuity from Teensy pins to the MOSFET gate/driver nets before enabling real outputs.
- Battery voltage sense uses D21.
- These assignments do not overlap. Firmware records this with
  BoardPinMap::PowerSense::conflictsWithPyroOutput, which must remain false for flight hardware.

Power / indicators
- Buzzer -> D2.
- Buzzer state feedback: INIT->SAFE seven medium-pitch tones; SAFE->ARMED 5 s
  continuous alert; all remaining transitions five quick tones.
- LED1 -> D34.
- LED2 -> D33.
- Battery voltage sense -> D21.
- Divider calibration: pack voltage / 5.545. At 12.6 V the ADC input is
  2.2723 V; at 11.1 V it is 2.0018 V.
- Firmware samples the divider with a 10-bit ADC. With a 3.3 V ADC reference,
  the expected codes are approximately 704 at 12.6 V and 620 at 11.1 V.
  Pack voltage is encoded to telemetry as FAST batt_mv in millivolts.

I2C0, BMP180 barometer
- Bus: Wire.
- SDA0 -> D18.
- SCL0 -> D19.
- Address: 0x77.
- MPL3115A2 is not used in the current firmware because the I2C0 barometer
  hardware is being isolated/replaced during bring-up.
- Recommended pull-ups: one SDA/SCL pull-up pair to 3.3 V on this bus if the PCB/module does not already provide it.

I2C1, LIS3MDL magnetometer
- Bus: Wire1.
- SDA1 -> D17.
- SCL1 -> D16.
- Address: 0x1C when DO/SDO is tied to GND.
- LIS CS -> 3.3 V to force I2C mode.
- LIS DO/SDO -> GND to select address 0x1C.
- Top/additional GND pad -> GND.
- Recommended pull-ups: one SDA/SCL pull-up pair to 3.3 V on this bus if the PCB/module does not already provide it.

Shared SPI0, LSM6DSO32 and H3LIS331DL
- SPI bus: SPI.
- MISO0 -> D12.
- MOSI0 -> D11.
- SCK0 -> D13.
- H3LIS331DL CS -> D0.
- LSM6DSO32 CS -> D10.
- Both sensors use 3.3 V logic and common GND.

GPS, GY-GPS6MV2 / u-blox M6
- UART: Serial3.
- GPS TX -> Teensy RX3 / D15.
- GPS RX -> Teensy TX3 / D14.
- Baud: 9600 by current firmware default.

LoRa, flight SX1262
- SPI data bus: SPI1 MISO D1, MOSI D26, SCK D27.
- SPI clock: 2 MHz, verified by the 2026-06-22 avionics radio bench test.
- NSS -> D9.
- RXE -> D30. This is not an SX1262 reset pin; its required polarity and TX/RX behavior must be confirmed from the PCB schematic before flight-radio enable.
- DIO1 -> D31.
- BUSY -> D32. SX1262 initialization requires this signal.
- No Teensy-controlled SX1262 `NRESET` net is currently recorded. The SX1262 HAL must use a no-reset configuration unless the schematic identifies another connection.

Pyro channels, firing order
- Pyro 1 / Drogue gpio1 -> D28.
- Pyro 1 / Drogue gpio2 -> D29.
- Pyro 1 / Drogue sense -> D25.
- Pyro 2 / Main output pair -> D35 and D38.
- Pyro 2 / Main gpio1/gpio2 order -> TODO: confirm by continuity from Teensy pins to the MOSFET gate/driver nets.
- Pyro 2 / Main sense -> D40.
- MOSFET HAL exists, but real GPIO output is dry-run unless `NURA_ENABLE_PYRO_OUTPUTS` is defined.
- The D21 voltage-sense input is distinct from both pyro channels. Before enabling
  real output, still LED/scope-test D28/D29/D35/D38 with no e-matches attached.

SPI1 / flight LoRa bus
- MISO1 -> D1.
- MOSI1 -> D26.
- SCK1 -> D27.
- SX1262 uses this bus; sensors remain on SPI0.

Expected quick bring-up observations
- I2C scanner should find BMP180 at 0x77 on Wire 18/19.
- I2C scanner should find LIS3MDL at 0x1C on Wire1 17/16.
- SPI0 WHOAMI/probe should use LSM6DSO32 CS D10 and H3LIS331DL CS D0.
- LoRa probe should use SPI1, NSS D9, RXE D30, DIO1 D31, and BUSY D32.
