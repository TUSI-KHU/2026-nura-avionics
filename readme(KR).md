# 2026-nura-avionics
2026 NURA 대회를 위한 임베디드 비행 제어기입니다.

## 개요
이 프로젝트는 스케줄러와 HAL 개념을 배우는 임베디드 시스템 입문자가 쉽게 접근할 수 있도록 설계되었습니다.

현재 포함된 내용:
- 협력형 태스크 스케줄러
- 기본 비행 상태 머신
- 비행 상태, 중단 상태, 센서 데이터를 위한 목적별 도메인 저장소
- 생성자 주입 방식의 태스크 의존성
- 보드 및 런타임 상수를 위한 인터페이스 기반 애플리케이션 설정
- 치명적 오류 처리를 위한 HAL 소유 panic 처리

## 왜 FreeRTOS를 사용하지 않나요?
FreeRTOS와 선점형 스케줄링은 강력하고 널리 사용됩니다. 하지만 우리는 초보자가 핵심 개념을 먼저 이해할 수 있도록 bare Arduino 위에서 단순한 스케줄러를 의도적으로 사용합니다.

현재 목표는 완벽한 양산급 비행 제어기를 만드는 것이 아니라, 다음과 같은 고급 임베디드 펌웨어 개념으로 이어지는 다리를 만드는 것입니다.
- 주기적 태스크 실행
- 결정론적 타이밍
- 명시적 상태 전이
- 작은 도메인 저장소를 통한 공유 데이터 흐름
- 논블로킹 흐름(2024 코드베이스 일부와 달리 `delay()` 대신 사용)

다음 단계에서는 검증되고 테스트된 RTOS를 반드시 다루게 될 예정이지만, 학습 경로상 그 시점은 조금 뒤입니다.

## 구조
- `app/`: composition root와 애플리케이션 설정.
- `core/`: 스케줄러, 로거, 복구 가능한 태스크 정책, 공통 태스크 인터페이스.
- `state/`: 비행 상태, 중단 상태, 센서 데이터를 위한 목적별 저장소.
- `missions/`: watchdog, logger drain, 비행 상태 머신(FSM) 같은 미션 수준 태스크.
- `sensors/`: 센서 획득 태스크.
- `hal/`: IMU 접근, 시리얼 로깅, 디지털 출력, panic 처리 같은 하드웨어 어댑터.
- `main.cpp`: `FlightControllerApp`으로 처리를 넘기는 얇은 Arduino 진입점.

## 런타임 모델
- `main.cpp`는 하나의 `FlightControllerApp`을 생성하고 `setup()`과 `loop()` 호출을 전달합니다.
- `FlightControllerApp`은 스케줄러, 저장소, HAL 객체, 설정, 태스크 인스턴스를 소유합니다.
- 태스크는 고정된 순서로 등록되며, 각 태스크의 `periodMs()`를 기준으로 `Scheduler`에 의해 협력적으로 실행됩니다.
- 공유 상태는 단일 전역 컨텍스트 객체 대신 작은 도메인 구조체에 저장됩니다.

## 설정
- `IAppConfig`는 시리얼 baud rate, IMU I2C 주소, 태스크 주기, 재시도 제한, panic LED 설정 같은 보드 및 런타임 상수를 정의합니다.
- `DefaultAppConfig`는 현재 보드 설정을 제공합니다.
- 태스크와 HAL 컴포넌트는 흩어진 리터럴 값 대신 생성자 주입을 통해 설정을 전달받습니다.

## 오류 처리
- 치명적 오류는 HAL이 소유한 `IPanicHandler` 구현체가 처리합니다.
- 현재 구현은 `BlinkingPanicHandler`를 사용해 설정된 상태 LED를 계속 깜빡입니다.

## 빌드 및 업로드


```bash
pio run -t upload
```

```bash
pio device monitor -b 115200
```
